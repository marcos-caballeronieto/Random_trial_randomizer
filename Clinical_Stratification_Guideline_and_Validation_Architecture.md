# Clinical Stratification Guidelines & Validation Architecture

**Document Type:** Clinical & Technical Specification  
**Classification:** Pre-Implementation Design  

## 1. Introduction & Core Concept
Stratified randomization guarantees that prespecified prognostic factors (covariates) are equally distributed between the treatment and control arms of a clinical trial. However, a common pitfall in clinical trial design is **Over-Stratification** (or *Strata Starvation*). While investigators often want to balance as many variables as possible, doing so arbitrarily can severely undermine the mathematical integrity of the permuted blocks.

This document establishes the clinical rationale for our engine's default variables, outlines the rules for optional variables, and provides a programmatic validation threshold to prevent model degradation.

---

## 2. Baseline Variables (Default Architecture)

By default, the engine enforces stratification on two fundamental biological covariates: **Biological Sex** and **Age Groups**.

### 2.1 Biological Sex (Dichotomous: Male / Female)
* **Clinical Rationale:** Men and women exhibit baseline differences in body composition (fat-to-muscle ratios), metabolic enzyme expression (e.g., Cytochrome P450 pathways), and renal clearance rates. These physiological differences directly affect drug pharmacokinetics (PK) and pharmacodynamics (PD). 
* **Failure Mode if Omitted:** An accidental concentration of one sex in a specific trial arm could mask adverse events or falsely suggest a lack of efficacy, introducing massive confounding bias.

### 2.2 Age Groups (Categorical / Binned)
* **Clinical Rationale:** Age is the most universal prognostic factor in medicine, strongly correlated with organ reserve capacity, arterial stiffness, and baseline mortality risk. 
* **Implementation Rule:** The engine rejects raw continuous age values. Continuous variables must be transformed into discrete clinical "bins" (e.g., `<40` [Young Adult], `40-65` [Middle Aged], `>65` [Geriatric/High Risk]). Stratifying by individual raw years (e.g., 42 vs. 43) would create an infinite number of strata, rendering block allocation impossible.

---

## 3. Optional Variables (Custom Comorbidities)

The system grants investigators the freedom to introduce custom categorical variables (e.g., specific diseases, smoking status, baseline disease severity scales). 

### 3.1 Clinical Selection Criteria
Investigators should **only** introduce an optional variable if it meets the following condition: **It must be an established, independent prognostic indicator of the primary trial outcome.**

| Optional Covariate Example | Clinical Context | Impact on Trial Validity |
| :--- | :--- | :--- |
| **Diabetes Status (Yes/No)** | Cardiovascular / Stroke trials | **Critical:** Diabetes dramatically alters vascular risk and recovery timelines. Must be balanced. |
| **Tumor Stage (I-II vs. III-IV)** | Oncology trials | **Critical:** Baseline tumor burden dictates survival rates regardless of therapy performance. |
| **Smoking Status (Yes/No)** | Respiratory / Pulmonary trials | **High Impact:** Directly affects lung compliance and drug metabolism in the lungs. |
| **Dietary Preference (Veg/Omni)** | Cardiovascular trial | **Low/None:** Unlikely to alter short-term pharmacological mechanisms. Avoid stratifying. |

---

## 4. The Mathematical Limit: Strata Starvation

The total number of strata ($K$) generated by the engine scales multiplicatively based on the number of levels ($c$) of each chosen covariate ($i$):

$$K = \prod_{i=1}^{m} c_i$$

If a trial investigator selects:
* Sex (2 levels)
* Age Group (3 levels)
* Diabetes (2 levels)
* Hypertension (2 levels)
* Smoking Status (2 levels)

The total number of distinct strata is: $2 \times 3 \times 2 \times 2 \times 2 = 48 \text{ unique strata}$.

If the total sample size ($N$) of the clinical trial is only 80 patients, the average number of patients per stratum will be $80 / 48 = 1.66$. 

### 4.1 Technical Consequence
Because Permuted Block Randomization requires a minimum number of patients to complete a block (typically a block size $B = 4$), **most strata will never capture enough patients to fill a single block.** The engine will fall back to simple unblocked randomization within those incomplete cells, completely failing to balance the groups.

## 5. Automated Validation Gatekeeper (Python Algorithm)

To protect investigators from silent design failures, the engine includes a pre-flight validation routine. This script calculates the structural viability of the requested stratification matrix before execution.

```python
import pandas as pd
import sys

def validate_stratification_matrix(df: pd.DataFrame, strata_columns: list, block_size: int = 4):
    """
    Validates if the sample size is sufficient to support the requested stratification strata.
    Lances a warning or terminates execution if Strata Starvation is detected.
    """
    total_patients = len(df)
    
    # Calculate the total number of unique combinations (Strata) present/expected
    # We count unique combinations in the actual dataset to see active fragmentation
    unique_strata_combinations = df[strata_columns].drop_duplicates()
    k_strata = len(unique_strata_combinations)
    
    # Theoretical cross-product maximum
    theoretical_k = 1
    for col in strata_columns:
        theoretical_k *= df[col].nunique()
        
    avg_patients_per_stratum = total_patients / k_strata
    
    print("="*60)
    print("CLINICAL TRIAL RANDOMIZATION ENGINE - PRE-FLIGHT VALIDATION")
    print("="*60)
    print(f"Total Enrolled Patients (N): {total_patients}")
    print(f"Selected Stratification Variables: {strata_columns}")
    print(f"Theoretical Max Strata Combinations: {theoretical_k}")
    print(f"Active Strata Observed in Dataset (K): {k_strata}")
    print(f"Configured Block Size (B): {block_size}")
    print(f"Average Patient Density per Stratum: {avg_patients_per_stratum:.2f}")
    print("-"*60)
    
    # CRITICAL THRESHOLD LOGIC:
    # Average density must be at least equal to 1.5 times the block size to ensure 
    # blocks can reasonably close and achieve true 1:1 numerical balance.
    critical_threshold = block_size * 1.5
    
    if avg_patients_per_stratum < block_size:
        print(f"CRITICAL WARNING: STRATA STARVATION DETECTED!")
        print(f"The average patient density ({avg_patients_per_stratum:.2f}) is lower than your block size ({block_size}).")
        print("Mathematical balance CANNOT be guaranteed. The algorithm will degrade to simple randomization.")
        print("RECOMMENDATION: Reduce the number of optional stratification variables or expand sample size.")
        print("="*60)
        return False
    elif avg_patients_per_stratum < critical_threshold:
        print(f"BORDERLINE WARNING: High risk of incomplete blocks at recruitment close.")
        print(f"Density ({avg_patients_per_stratum:.2f}) is close to minimum limits.")
        print("Proceed with caution. Minor imbalances may occur in rare strata.")
        print("="*60)
        return True
    else:
        print("VALIDATION SUCCESSFUL: Stratification matrix is structurally sound and balanced.")
        print("="*60)
        return True

# --- Example Usage Run ---
if __name__ == "__main__":
    # Simulating a high-risk scenario: Small sample size, too many variables
    data = {
        'Patient_ID': [f"PAC-{i:03d}" for i in range(60)],
        'Sex': ['M', 'F'] * 30,
        'Age_Group': ['Young', 'Middle', 'Elderly'] * 20,
        'Diabetes': ['Yes', 'No', 'No'] * 20,
        'Hypertension': ['Yes', 'Yes', 'No', 'No'] * 15
    }
    df_trial = pd.DataFrame(data)
    
    # Testing a safe configuration (Only Sex and Age Group) -> 6 strata for 60 patients = 10 p/s (Safe)
    validate_stratification_matrix(df_trial, strata_columns=['Sex', 'Age_Group'], block_size=4)
    
    # Testing an unsafe configuration (Adding Diabetes and Hypertension) -> Over-fragmentation
    validate_stratification_matrix(df_trial, strata_columns=['Sex', 'Age_Group', 'Diabetes', 'Hypertension'], block_size=4)
```
